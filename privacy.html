Privacy Policy

How EnQMail collects, uses, stores and protects information.

Last Updated: 21 June 2026

1. Introduction

EnQMail is an email organization and review application. This Privacy Policy explains how information is collected and processed when using EnQMail.

2. Data Controller

Developer: Prasad Mahankal

Contact: enqmail27@gmail.com

3. Information We Collect

Account Information

  • Name
  • Email address
  • Google Account ID
  • Google profile photo URL
  • Internal user identifier

Gmail Data

  • Email subject lines
  • Sender and recipient information
  • Email body content
  • Email timestamps
  • Email metadata
  • Gmail labels

EnQMail stores synchronized email bodies in its database to provide Review Queue, My Emails, folders, reminders, and email organization features.

Attachment Metadata

EnQMail does not download or store attachment files. It may store attachment metadata such as filename, file size, and MIME type.

User Organization Data

  • Folder names
  • Folder assignments
  • Review Queue status
  • Needs Action status
  • Save For Reference status
  • Reminder settings
  • Routing rules
  • Blocked sender rules
  • Review markers

4. How We Use Information

  • Authenticate users
  • Synchronize Gmail accounts
  • Provide Review Queue functionality
  • Organize emails into folders and categories
  • Create reminders
  • Store user preferences and workflow state
  • Maintain application security and reliability

5. Gmail Access

EnQMail requests the Google OAuth scope:

https://www.googleapis.com/auth/gmail.modify

This scope is used to read Gmail messages and modify Gmail message state, including read/unread status and labels as part of EnQMail's review and organization workflow.

6. Google API Services User Data Policy

EnQMail's use and transfer of information received from Google APIs adheres to the Google API Services User Data Policy, including the Limited Use requirements.

  • Google user data is not sold.
  • Google user data is not used for advertising purposes.
  • Google user data is used only to provide EnQMail functionality.
  • Google user data is not used to train generalized AI or machine learning models.

7. Storage & Security

Server-side data is stored in Neon PostgreSQL.

Authentication is provided using Firebase Authentication and Google OAuth.

Data is transmitted using encrypted HTTPS connections.

Local Device Storage

EnQMail may store the following locally using AsyncStorage:

  • Authentication/session information
  • Offline mode configuration
  • API configuration
  • Reminder notification identifiers
  • Offline-mode email and workflow data

8. Notifications

EnQMail uses local device notifications for email reminders created by users. Remote push notifications are not currently used.

10. Account Deletion

EnQMail provides in-app account deletion through:

Settings & Rules → Privacy & Safety → Delete My Account & Data

When an account is deleted, the user record and associated emails, folders, routing rules, reminders, and related data are deleted immediately.

11. Your Rights

You may contact enqmail27@gmail.com regarding privacy questions, data access requests, or account-related concerns.

12. Children's Privacy

EnQMail is not intended for children under 13 years of age.

13. Changes

This Privacy Policy may be updated from time to time. The updated version will display a revised effective date.
